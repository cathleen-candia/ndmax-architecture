Data Tables API — Key Endpoints
POST Create Data Table
{dtBase}/datatables/{repoId}/create-table
Creates table with column schema. Run once per entity type on first activation. Check existence with GET first.
POST Upsert Row (preferred)
{dtBase}/datatables/{repoId}/insert-row
Insert or update by uniqueKey. Prevents duplicates on entity updates. Primary write method.
GET List Data Tables
{dtBase}/datatables/{repoId}
Check if CB tables already exist before creating. Also used for target table selection in workflow config UI.
GET Get Table Data
{dtBase}/datatables/{repoId}/{tableId}/data
Read rows back for verification. Used in sync status dashboard and E2E testing.
GET Get Table Schema
{dtBase}/datatables/{repoId}/{tableId}/schema
Retrieve column definitions. Use to validate mapping config matches actual table structure.
PUT Update Table Schema
{dtBase}/datatables/{repoId}/{tableId}/update-table
Modify columns when field mapping changes after initial setup. Handle schema migration gracefully.

Unauthorized
→ refresh token
→ retry 1x
→ log if still fails
429
Rate Limited
→ backoff exponential
→ retry 3x
→ queue if exhausted
5xx

→ retry 3x w/ backoff
→ queue 5m/30m/2h
→ alert admin panel
NET
Timeout
→ retry 1x
→ queue for later
→ never block CB save
