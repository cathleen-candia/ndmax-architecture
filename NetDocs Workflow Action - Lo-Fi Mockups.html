Lo-Fi Mockups: "Send to NetDocs Data Table" Workflow Action

5 screens showing how the new NetDocs integration fits into the existing Centerbase workflow builder. Based on the Basic & Advanced Workflow Workshop documentation.

Design Principle

The new "Send to NetDocs Data Table" action follows the same patterns as existing Centerbase workflow actions (User Action, Automatic Action, Update Command, Create Command). An admin adds it from the same gear menu, configures it in the same form layout, and it appears in the Workflow Details view using the same node/action hierarchy.

What's New

A new action type ("Send to NetDocs Data Table") and a new field mapping configuration panel that lets admins pick Centerbase fields and map them to NetDocs data table columns. The new elements use a purple accent to visually distinguish NetDocs-specific features from native Centerbase actions.

Screens Included

1 Create Action form with new action type
2 Field mapping configuration panel
3 Workflow Details view with NetDocs action
4 Sync status dashboard (admin settings)
5 Error handling & retry panel

5 Error Details & Retry Panel

When an admin clicks "Details" on a failed sync, they see a plain-language error description with the option to retry. Maps to Jira ticket CC-47638's requirement for "plain-language descriptions (not raw API errors)."

What happened

Centerbase was unable to send matter data to the NetDocs data table. The NetDocs server returned a "service unavailable" error, which usually means NetDocs is experiencing temporary downtime or maintenance.

What to do

Click "Retry Now" to attempt the sync again. If this keeps failing, check the NetDocs status page or contact your NetDocs administrator.

Design note: Error messages are translated into plain language for firm admins — they should never see raw API responses. The retry history gives visibility into what's been attempted. The "Entity saved in Centerbase?" confirmation is critical for user trust: admins need to know that a sync failure did not cause data loss in their own system.
